01 / FOUNDATIONS

From Zero to AI-Ready

Understand the landscape before you build. This section covers the mental models, vocabulary, and core skills every AI practitioner needs.

Core Concepts You Must Know

🧠

Large Language Models

Transformer-based models trained on vast text corpora. Key insight: LLMs are next-token predictors, not knowledge databases. Learn prompting before fine-tuning.

🤖

AI Agents

Agents are LLMs given tools, memory, and a feedback loop. They observe → reason → act → observe. The core loop is what makes them autonomous.

⚡

Automation vs AI

Automation = deterministic rules. AI = probabilistic reasoning. Modern systems combine both. Know when to use a rule and when to use a model.

🔌

APIs & Webhooks

Every AI system talks through APIs (request/response) or webhooks (event-push). Master REST API calls and you can connect anything to anything.

🔄

Workflows

A workflow is a directed graph of steps. Trigger → Process → Output. Tools like n8n and Make visualize these as flowcharts you can build without code.

📝

Prompt Engineering

The art of crafting inputs to get reliable, structured outputs from LLMs. Mastery here replaces hundreds of lines of NLP code from 5 years ago.

Prompt Engineering Fundamentals

# The anatomy of a production-grade prompt SYSTEM: You are a senior customer support agent for Acme Corp. Respond in JSON only. Be concise and professional. CONTEXT (inject dynamically): Customer tier: {tier} Past interactions: {history} TASK: Classify the following support ticket into: - category: [billing | technical | general] - priority: [low | medium | high | urgent] - sentiment: [positive | neutral | negative] - suggested_action: string TICKET: {user_message} RULES: - Never mention competitor products - Escalate if priority = urgent - Return ONLY valid JSON, no prose
✦ Pro Tip: Chain of Thought

Add "Think step by step before answering" to complex reasoning tasks. This alone improves accuracy by 30–40% on multi-step problems. Use <thinking> tags to separate reasoning from output.

Learning Timeline — Foundations

Week 1–2

AI Mental Models

What are LLMs, tokens, embeddings, context windows. Play with ChatGPT and Claude API. Read "Attention Is All You Need" summary.

Week 3–4

APIs & HTTP

Make your first OpenAI API call in Python. Understand request headers, JSON bodies, error codes. Build a simple chatbot in 30 lines of code.

Week 5–6

Prompt Engineering Mastery

System prompts, few-shot examples, chain-of-thought, output formatting, structured JSON extraction. Mini-project: build a sentiment classifier.

Week 7–8

Minimal Coding Foundation

Python: variables, functions, loops, dicts, requests library. JavaScript: async/await, fetch, JSON. You don't need to be a developer — you need to read and modify code.

⚠ Common Mistake #1

Skipping foundations to jump into agents. If you don't understand how a single LLM call works, you cannot debug a multi-agent system. Invest 2 weeks here — it pays dividends forever.

Event-Driven Architecture Pattern

✦ Core Pattern: Webhook-Driven Systems

Build all automations on this principle: Event → Queue → Process → Act → Log. Every incoming webhook is an event. Queue it (to avoid loss), process it (AI/logic), act on it (write to DB, send message), log it (Airtable/Supabase). This pattern scales from 10 to 10,000 events/day without changes.

03 / AI ENGINEER

Practical AI Engineering

RAG systems, vector databases, embeddings, and memory — the building blocks of production AI applications that actually work.

🔍

RAG (Retrieval-Augmented Generation)

Inject external knowledge into LLM context at inference time. The solution to hallucination for domain-specific applications.

📊

Embeddings

Convert text to high-dimensional vectors that encode semantic meaning. Similar meanings → similar vectors. Foundation of search & RAG.

🗄️

Vector Databases

Pinecone, Weaviate, ChromaDB, Qdrant — stores and retrieves vectors via similarity search. The memory layer of your AI system.

🧩

Memory Systems

Short-term (conversation buffer), long-term (vector DB), entity memory (key facts), summary memory (compressed history). LangChain implements all four.

Project 3: Document Q&A System (RAG)

RAG Pipeline Architecture
INGESTION: PDF / Docs ──► Chunking (512 tokens) ──► Embeddings (ada-002) ──► Pinecone / Chroma
────────────────────────────
QUERY: User Question ──► Embed query ──► Similarity Search (top-k)
──► Prompt + Context + Question ──► GPT-4o Answer
# Minimal RAG in Python (LangChain) from langchain.document_loaders import PyPDFLoader from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain.embeddings import OpenAIEmbeddings from langchain.vectorstores import Chroma from langchain.chat_models import ChatOpenAI from langchain.chains import RetrievalQA # 1. Load & chunk document loader = PyPDFLoader("company_policy.pdf") docs = loader.load() splitter = RecursiveCharacterTextSplitter(chunk_size=512, chunk_overlap=50) chunks = splitter.split_documents(docs) # 2. Embed & store vectorstore = Chroma.from_documents(chunks, OpenAIEmbeddings()) # 3. Query qa = RetrievalQA.from_chain_type( llm=ChatOpenAI(model="gpt-4o"), retriever=vectorstore.as_retriever(search_kwargs={"k": 4}) ) answer = qa.run("What is the remote work policy?")

from crewai import Agent, Task, Crew, Process # Define specialized agents researcher = Agent( role='Market Research Analyst', goal='Find competitor pricing and positioning', tools=[search_tool, scrape_tool], llm='gpt-4o' ) strategist = Agent( role='Business Strategist', goal='Synthesize research into actionable strategy', llm='gpt-4o' ) writer = Agent( role='Content Writer', goal='Write compelling proposal from strategy', llm='gpt-4o' ) # Sequential pipeline crew = Crew( agents=[researcher, strategist, writer], tasks=[research_task, strategy_task, write_task], process=Process.sequential ) result = crew.kickoff()

⚠ Common Mistake #2: Infinite Agent Loops

Always set a maximum iteration limit on agents. Without it, a confused agent can loop endlessly and rack up API costs. Also: always have a human-in-the-loop for high-stakes actions like sending emails or making purchases.

05 / ARCHITECT LEVEL

AI Systems Architecture

Design systems that scale, survive failures, protect data, and stay economically viable. The difference between a demo and a product.

🏗️

Hybrid LCNC + Code

Use LCNC for orchestration (n8n), code for custom logic (Python), APIs for AI (OpenAI), managed DBs for data (Supabase). Each layer does what it does best.

📈

Scaling Patterns

Queue-based decoupling (BullMQ, Redis), async processing, rate limiting, caching LLM responses with semantic similarity (saves 40-70% cost).

🔒

Security First

API keys in env vars, never in code. Prompt injection defenses. PII redaction before sending to LLMs. Output validation before executing agent actions.

💰

Cost Optimization

Route simple tasks to GPT-4o-mini, complex to GPT-4o. Cache identical requests. Compress context. Use local models (Ollama) for internal/dev workloads.

Reference Architecture: Production AI SaaS

Layered Architecture
CLIENTS Web App | Mobile App | WhatsApp | Slack Bot
GATEWAY API Gateway + Auth (Supabase Auth) ──► Rate Limiter
ORCH n8n Orchestrator ──► AI Router ──► gpt-4o / claude / gemini
DATA Supabase (Postgres) + Pinecone (Vectors) + Redis (Cache)
OBS LangSmith (LLM traces) + Sentry (errors) + Grafana (metrics)
✦ Architect's Golden Rule

Design for failure first. Every external API call (LLM, database, webhook) can fail. Implement retries with exponential backoff, dead-letter queues for failed jobs, and fallback responses. A system that fails gracefully is more valuable than one that never fails in development but collapses in production.

Common Mistakes to Avoid

🚫

Tool Obsession

Switching tools every week. Pick one stack, build 3 real projects with it, then evaluate alternatives. Depth beats breadth at every career stage.

🚫

Building in Secret

Waiting until your project is "perfect" before sharing. Document your process in public. Clients hire based on demonstrated thinking, not polished demos.

🚫

Ignoring Business Context

Technical skill without business understanding = developer. Business understanding with technical skill = consultant who charges 5x more.
